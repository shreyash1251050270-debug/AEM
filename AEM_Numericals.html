📌 Quick Reference — Winding Types

  • Wave winding: A = 2 (always 2 parallel paths, regardless of poles)
  • Lap winding: A = P (parallel paths equal to number of poles)
  • Shunt motor: Field winding in parallel → I = Ia + Ish
  • Series motor: Field winding in series → I = Ia = Ise

💡 Exam Tips — DC Motor Numericals

  • First step always: Identify motor type (Shunt/Series) and winding type (Wave/Lap) — this determines A and current equations.
  • For Shunt motor: Always calculate Ish first, then Ia = I − Ish. Never assume Ia = I!
  • Wave winding → A = 2. Lap winding → A = P. This is the most frequently tested fact.
  • Verify your answer by cross-checking using the Power method: P = Eb × Ia, then T = P/ω.
  • ⚠️ Common mistake: Forgetting to subtract Ish from total current I to get armature current Ia.
Numerical 01 EASY 5 Marks

A DC motor connected to 450V supply takes 120A on full load. Ra = 0.15Ω, Rsh = 150Ω. Calculate back EMF.

Given Data

Given
V = 450 V (Supply voltage)
I = 120 A (Total current on full load)
Ra = 0.15 Ω (Armature resistance)
Rsh = 150 Ω (Field resistance)

Step 1: Find Shunt Field Current (Ish)

Formula Ish = V / Rsh Ish = 450 / 150 Ish = 3 A

Step 2: Find Armature Current (Ia)

Formula Ia = I − Ish Ia = 120 − 3 Ia = 117 A

Step 3: Calculate Back EMF (Eb)

Formula Eb = V − Ia × Ra Eb = 450 − (117 × 0.15) Eb = 450 − 17.55 Eb = 432.45 V ✓
✅ Answer
Back EMF at full load = 432.45 V
Numerical 02 HARD — MULTI-PART 5 Marks

A 4-pole, 500V DC shunt motor takes Ia = 80A. Ra = 0.4Ω. Wave wound, Z = 522 conductors, Φ = 0.025 Wb. Find (a) Back EMF (b) Speed (c) Torque.

Given Data

Given
P = 4 poles
V = 500 V
Ia = 80 A (Armature current)
Ra = 0.4 Ω
Z = 522 conductors
Φ = 0.025 Wb (flux per pole)
Winding: Wave wound → A = 2

(a) Back EMF (Eb)

Step 1 Eb = V − Ia × Ra Eb = 500 − (80 × 0.4) Eb = 500 − 32 Eb = 468 V

(b) Speed of Motor (N)

Step 2 Using: Eb = (Φ × Z × N × P) / (60 × A) Rearranging for N: N = (Eb × 60 × A) / (Φ × Z × P) N = (468 × 60 × 2) / (0.025 × 522 × 4) N = 56,160 / 52.2 N ≈ 1076 RPM

(c) Torque Developed (T)

Step 3 — Method 1 (Direct Formula) T = (Φ × Z × Ia × P) / (2π × A) T = (0.025 × 522 × 80 × 4) / (2 × 3.14159 × 2) T = 4176 / 12.566 T = 332.27 N·m Verification — Method 2 (Power Method) P = Eb × Ia = 468 × 80 = 37,440 W ω = 2πN/60 = 2π × 1076/60 = 112.68 rad/s T = P / ω = 37440 / 112.68 T = 332.27 N·m ✓ (Verified!)
✅ Answers
ParameterValue
(a) Back EMF468 V
(b) Speed1076 RPM
(c) Torque332.27 N·m

Practice Numerical 03 MEDIUM 5 Marks

A 6-pole DC shunt motor has a lap winding with 600 conductors. Supply voltage = 240V, Ia = 50A, Ra = 0.3Ω, Φ = 0.02 Wb. Find back EMF and speed.

Given Data

Given
P = 6 poles
V = 240 V
Ia = 50 A
Ra = 0.3 Ω
Z = 600 conductors
Φ = 0.02 Wb
Winding: Lap wound → A = P = 6

Step 1: Back EMF

Formula Eb = V − Ia × Ra Eb = 240 − (50 × 0.3) Eb = 240 − 15 Eb = 225 V

Step 2: Speed

Formula N = (Eb × 60 × A) / (Φ × Z × P) N = (225 × 60 × 6) / (0.02 × 600 × 6) N = 81,000 / 72 N = 1125 RPM
✅ Answers
Eb = 225 V, N = 1125 RPM

💡 Key Insight — Lap vs Wave

  • Notice: For lap winding, A = P = 6. This is the critical difference from wave winding where A = 2.
  • If this same problem had wave winding, speed would be N = (225 × 60 × 2) / (0.02 × 600 × 6) = 27,000 / 72 = 375 RPM — much slower!
  • This shows why winding type drastically affects speed.

Formula

Master Formula Fgrip = m × (g + a) × FOS / (μ × n) Where: m = mass of object (kg) g = 9.81 m/s² a = acceleration (m/s²), use 0 if stationary FOS = Factor of Safety (default = 1 if not given) μ = coefficient of friction n = number of gripper fingers

💡 Exam Tips — Gripper Numericals

  • Always check: Is there acceleration? If yes, use F = m(g + a). If only gravity, use F = mg.
  • Factor of Safety: Only apply FOS if explicitly given. Multiply it with the weight, NOT with the final answer.
  • Units matter: Convert piston diameter from mm to m before calculating area. 40 mm = 0.04 m.
  • Pressure conversion: 1 bar = 100,000 Pa = 100 kPa. 1 psi = 6895 Pa.
  • ⚠️ Common mistake: Fgrip is per finger. Don't forget to divide by n (number of fingers)!
  • ⚠️ Common mistake: Using acceleration but forgetting to add g. The total acceleration is always (g + a), not just a.
Numerical 01 EASY 5 Marks

A gripper lifts a 4 kg object upward with acceleration 2 m/s². μ = 0.4, n = 2 fingers. Find gripping force per finger.

Given Data

Given
m = 4 kg
a = 2 m/s² (upward acceleration)
μ = 0.4 (coefficient of friction)
n = 2 fingers
g = 9.81 m/s²

Step 1: Total Force on Object

Key: Object is accelerating upward Ftotal = m × (g + a) Ftotal = 4 × (9.81 + 2) Ftotal = 4 × 11.81 Ftotal = 47.24 N

Step 2: Gripping Force per Finger

Friction equilibrium: Ftotal = Fgrip × μ × n Fgrip = Ftotal / (μ × n) Fgrip = 47.24 / (0.4 × 2) Fgrip = 47.24 / 0.8 Fgrip = 59.05 N per finger
✅ Answer
Gripping force per finger = 59.05 N
Numerical 02 MEDIUM — WITH FOS 5 Marks

10 kg workpiece handled by a 3-finger gripper. μ = 0.3, Factor of Safety = 2. Calculate gripping force per finger.

Given Data

Given
m = 10 kg
n = 3 fingers
μ = 0.3
FOS = 2
g = 9.81 m/s²
Note: No acceleration mentioned → a = 0

Step 1: Weight of Workpiece

Weight W = m × g W = 10 × 9.81 W = 98.1 N

Step 2: Apply Factor of Safety

Design Force Fdesign = W × FOS Fdesign = 98.1 × 2 Fdesign = 196.2 N

Step 3: Gripping Force per Finger

Formula Fgrip = Fdesign / (μ × n) Fgrip = 196.2 / (0.3 × 3) Fgrip = 196.2 / 0.9 Fgrip = 218 N per finger
✅ Answer
Gripping force per finger = 218 N
Numerical 03 HARD — PNEUMATIC 5 Marks

A pneumatic gripper lifts 6 kg object, 2 fingers. μ = 0.5, piston diameter = 40 mm. Find required air pressure.

Given Data

Given
m = 6 kg
n = 2 fingers
μ = 0.5
d = 40 mm = 0.04 m
g = 9.81 m/s²

Step 1: Weight of Object

Weight W = m × g = 6 × 9.81 W = 58.86 N

Step 2: Gripping Force per Finger

Grip Force Fgrip = W / (μ × n) Fgrip = 58.86 / (0.5 × 2) Fgrip = 58.86 / 1.0 Fgrip = 58.86 N per finger

Step 3: Piston Area

Area of circular piston A = π/4 × d² A = (3.14159/4) × (0.04)² A = 0.7854 × 0.0016 A = 1.2566 × 10⁻³ m²

Step 4: Required Air Pressure

Pressure = Force / Area P = Fgrip / A P = 58.86 / (1.2566 × 10⁻³) P = 46,840 Pa P ≈ 46.84 kPa ≈ 0.468 bar ≈ 6.8 psi
✅ Answer
Air pressure required = 46.84 kPa (≈ 0.468 bar ≈ 6.8 psi)

💡 Pro Tip — Unit Conversions for Pressure

  • Always convert diameter to meters before calculating area. 40 mm = 0.04 m, NOT 40.
  • Final answer in kPa is most common in exams. Divide Pa by 1000.
  • If asked in bar: divide kPa by 100. If asked in psi: multiply bar by 14.5.
🎯 Exam shortcut: For d = 40mm, remember A ≈ 1.257 × 10⁻³ m². For d = 50mm, A ≈ 1.963 × 10⁻³ m².

BJT Transistor Numericals

📐 Transistor — Key Formulas

Formula
Description
Ie = Ic + Ib
KCL at transistor (always true)
α = Ic / Ie
Common Base current gain (< 1)
β = Ic / Ib
Common Emitter current gain (> 1)
γ = Ie / Ib
Common Collector current gain
α = β / (β + 1)
α in terms of β
β = α / (1 − α)
β in terms of α
γ = β + 1 = 1/(1 − α)
γ in terms of β or α
Memory Aid — The Holy Trinity Ie = Ic + Ib ← Start here ALWAYS α = Ic/Ie → α is always < 1 (0.95–0.99) β = Ic/Ib → β is always > 1 (20–500) γ = Ie/Ib → γ = β + 1 Quick check: α + (1/γ) must never exceed 1

💡 Exam Tips — Transistor Numericals

  • Three types of problems: Given (β, Ib), Given (Ie, Ic), Given (α, Ic). Identify which type and pick the right starting formula.
  • Always find Ic first if not given. Everything else follows from Ic.
  • Cross-verify α: Calculate using both α = Ic/Ie AND α = β/(β+1). Both must match.
  • Unit conversion: μA = 10⁻⁶ A, mA = 10⁻³ A. Keep consistent units throughout.
  • ⚠️ Common mistake: Writing Ie = Ic − Ib (WRONG!). It's always Ie = Ic + Ib.
  • ⚠️ Common mistake: Confusing α and β. Remember: α < 1, β > 1. If your answer has α > 1, you've swapped them!
🎯 Fastest approach: For any problem, write Ie = Ic + Ib first. Then compute whatever is asked using the three ratios.
Numerical 01 EASY 5 Marks

Given: β = 75, Ib = 40 μA (CE configuration). Find Ic, Ie, α and γ.

Given Data

Given
β = 75
Ib = 40 μA = 40 × 10⁻⁶ A

Step 1: Collector Current (Ic)

From β definition Ic = β × Ib Ic = 75 × 40 μA Ic = 3000 μA = 3 mA

Step 2: Emitter Current (Ie)

KCL Ie = Ic + Ib Ie = 3000 + 40 Ie = 3040 μA = 3.04 mA

Step 3: Alpha (α)

Two methods (both must match) Method 1: α = Ic / Ie = 3000/3040 = 0.9868 Method 2: α = β/(β+1) = 75/76 = 0.9868 ✓

Step 4: Gamma (γ)

Two methods Method 1: γ = Ie / Ib = 3040/40 = 76 Method 2: γ = β + 1 = 75 + 1 = 76 ✓
✅ Answers

Given Data

Given

Numerical 03 MEDIUM 5 Marks

Given: α = 0.995, Ic = 2A (Common Base). Find Ie, Ib and β.

Given Data

Given
α = 0.995
Ic = 2 A

Step 1: Emitter Current (Ie)

From α definition α = Ic / Ie Ie = Ic / α Ie = 2 / 0.995 Ie = 2.01 A

Step 2: Base Current (Ib)

KCL Ib = Ie − Ic Ib = 2.01 − 2 Ib = 0.01 A = 10 mA

Step 3: Beta (β)

Two methods Method 1: β = Ic / Ib = 2/0.01 = 200 Method 2: β = α/(1−α) = 0.995/0.005 = 199 ≈ 200 ✓ Note: Small rounding difference is acceptable in exams
✅ Answers
ParameterValue
Ie2.01 A
Ib0.01 A (10 mA)
β200

💡 Key Insight — High α means High β

  • When α = 0.995 (very close to 1), β = 200 (very high). This is a good quality transistor.
  • When α = 0.96, β = 24 (moderate). Lower quality but still usable.
  • A small increase in α causes a huge increase in β. That's why high-α transistors are preferred.

📐 ATmega328P Timer & PWM — Key Formulas

Formula
Description
ftimer = fclk / Prescaler

Ttick = 1 / ftimer

Timer0/2: Max = 256 (8-bit)

📌 Quick Reference — Prescaler Values

  • ATmega328P clock: fclk = 16 MHz (Arduino Uno)
  • Available prescalers: 1, 8, 64, 256, 1024
  • Timer0: 8-bit (0–255), Timer1: 16-bit (0–65535), Timer2: 8-bit (0–255)
  • ADC resolution: 10-bit → 1024 levels (0 to 1023)

💡 Exam Tips — Timer & PWM Calculations

  • Approach for "generate X ms delay": Choose prescaler → calculate Ttick → find counts needed → check if fits in timer (8-bit: ≤256, 16-bit: ≤65536).
  • For PWM: Duty cycle = OCR/TOP × 100%. If TOP = 255 (8-bit fast PWM), OCR = 127 gives 50% duty.
  • ADC shortcut: For 5V ref, each step = 5/1024 ≈ 4.88 mV. So ADC reading × 4.88mV = voltage.
  • ⚠️ Common mistake: Using 1023 instead of 1024 for ADC calculations. The ADC has 1024 levels (0–1023), but the divisor is 1024.
Numerical 01 MEDIUM 5 Marks

ATmega328P runs at 16 MHz. Using Timer1 (16-bit) with prescaler = 64, calculate (a) Timer tick period, (b) Maximum delay possible before overflow.

Given Data

Given
fclk = 16 MHz = 16 × 10⁶ Hz
Prescaler = 64
Timer1 = 16-bit → Max count = 65536

(a) Timer Tick Period

Step 1: Timer frequency ftimer = fclk / Prescaler ftimer = 16,000,000 / 64 ftimer = 250,000 Hz = 250 kHz Step 2: Tick period Ttick = 1 / ftimer Ttick = 1 / 250,000 Ttick = 4 μs (4 microseconds)

(b) Maximum Delay (Overflow Time)

Maximum time before overflow Toverflow = Max count × Ttick Toverflow = 65536 × 4 μs Toverflow = 262,144 μs Toverflow ≈ 262.14 ms ≈ 0.262 seconds
✅ Answers
ParameterValue
(a) Timer tick period4 μs
(b) Max delay (overflow)262.14 ms
Numerical 02 EASY 5 Marks

Using Timer0 in Fast PWM mode (8-bit, TOP = 255), fclk = 16 MHz, prescaler = 64. Find (a) PWM frequency, (b) OCR value for 75% duty cycle.

Given Data

Given
fclk = 16 MHz
Prescaler = 64
TOP = 255 (8-bit fast PWM)
Duty cycle = 75%

(a) PWM Frequency

Fast PWM frequency formula fPWM = fclk / (Prescaler × (TOP + 1)) fPWM = 16,000,000 / (64 × 256) fPWM = 16,000,000 / 16,384 fPWM ≈ 976.56 Hz ≈ 977 Hz

(b) OCR for 75% Duty Cycle

Duty cycle formula Duty% = (OCR + 1) / (TOP + 1) × 100 0.75 = (OCR + 1) / 256 OCR + 1 = 0.75 × 256 = 192 OCR = 191 Verify: (191+1)/256 × 100 = 75% ✓

Numerical 03 EASY 5 Marks

ATmega328P ADC has 10-bit resolution with Vref = 5V. (a) What is the ADC resolution in mV? (b) If ADC reads 512, what is the input voltage? (c) What ADC value corresponds to 3.3V input?

Given Data

Given
ADC = 10-bit → 1024 levels (0 to 1023)
Vref = 5V

(a) ADC Resolution

Voltage per step Resolution = Vref / 210 = 5 / 1024 Resolution = 4.883 mV per step

(b) ADC Value 512 → Voltage

Convert digital to analog Vin = (ADC / 1024) × Vref Vin = (512 / 1024) × 5 Vin = 0.5 × 5 Vin = 2.5 V

(c) 3.3V → ADC Value

Convert analog to digital ADC = (Vin / Vref) × 1024 ADC = (3.3 / 5) × 1024 ADC = 0.66 × 1024 ADC ≈ 676
✅ Answers
PartAnswer
(a) Resolution4.883 mV/step
(b) V at ADC=5122.5 V
(c) ADC at 3.3V676

💡 ADC Quick Shortcuts

  • ADC = 0 → 0V, ADC = 512 → 2.5V (half), ADC = 1023 → ~5V
  • Each step ≈ 4.88 mV for 5V reference. Multiply ADC value by ~0.00488 for quick conversion.
  • Arduino's analogRead() returns 0–1023. Use voltage = analogRead(pin) * 5.0 / 1024;

🎯 Final Exam Strategy — Numericals

  • Read the problem twice. Identify all given values and note what's asked.
  • Write all given data first. This earns marks even if the calculation goes wrong.
  • Write the formula before substituting. Examiners award marks for correct formula.
  • Show every step. Never skip — each step can earn partial marks.
  • Box your final answer. Make it clearly visible.
  • Include units. V, A, N, RPM, N·m, Pa, kPa — always write units with answers.
  • Cross-verify using an alternative method when possible (especially DC Motor torque and transistor α).
