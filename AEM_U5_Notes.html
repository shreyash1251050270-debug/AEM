01 — Foundations

Computing Basics & Flip-Flops

What does "Compute" mean?

To compute means to count and to calculate. Every computing device at its core must handle:

  • Input data — receiving data from the external world
  • Processing — performing operations on the data (arithmetic, logic)
  • Output — producing a result/answer

Additional features expected of a modern computing device:

  • Storage of data and output for future retrieval
  • Print / Display / Transfer output
  • Ability to modify/update the process (reprogrammability)

Role of Flip-Flops

A flip-flop is the fundamental building block of digital memory:

What it stores
1 single bit (binary digit) — either 0 or 1
Stable states
Two stable states → called a Bistable Multivibrator
Purpose
Data storage and retrieval — the basis of all memory
Scaling
Many flip-flops together form the complete memory of a computing device
Types used
Flip-flops and Latches are both used as data storage elements
Key insight A register in a CPU is just a group of flip-flops. 8 flip-flops = 1 byte register. 32 flip-flops = 32-bit register.
02 — Core Hardware

Microprocessor vs. Microcontroller

Microprocessor (µP)

An electronic device that is a Central Processing Unit (CPU) on a single integrated circuit chip. It contains millions of transistors, resistors, and diodes.

  • Has only the CPU inside — no ROM, RAM, or I/O built-in
  • All memory and peripherals must be added externally
  • No specific task — fully programmable as per requirement
  • Results in a large PCB with many external chips

Microcontroller (µC)

A microcontroller has the CPU plus all the following embedded on a single chip:

03 — Embedded Systems

Embedded Systems

An Embedded System is a combination of hardware and software designed for a specific dedicated task. The firmware (specific program) is written to ROM and cannot be used in any other system.

Key Features

  • Combination of H/W and S/W
  • Specific task given — not general purpose
  • Dialogue between hardware and software
  • Can be a small part of a bigger system
  • Requires less power
  • Highly stable, reliable, efficient
  • Minimal user interface
  • Time specific & task specific
  • Mass produced → very cheap
  • User friendly
  • Development creates obsolescence of older version
  • Examples: Washing machines, Mobile phones, Microwave ovens, Automobiles, Video devices, Games, Music systems, Cameras, Medical instruments, Defense weapons

Classification of Embedded Systems

By Scale / Complexity

By Application Type

Least complicated. Works independently without a host computer. Input can be analog or digital. e.g. Calculator, Digital wristwatch
Real-time
Functions within a definite time interval. Soft real-time (Microwave, Washing machine) — deadline can be missed occasionally. Hard real-time (Missiles, Robotics, Space) — deadline missing is catastrophic.
Networked

Mobile
Portable embedded systems for cell phones, PDAs, digital data devices

Embedded System Hardware Approaches

Microcontroller-based

  • CPU + ROM + RAM + I/O on single chip
  • Simple programming, low cost
  • ATmega328P, 8051, PIC, STM32

FPGA / ASIC-based

  • Field Programmable Gate Array
  • Application-Specific Integrated Circuit
  • Used for high-speed, custom hardware

DDR / PORT / PIN Registers (for each Port)

Every port has three 8-bit registers controlling its behavior:

DDRx
Data Direction Register — set bit HIGH = Output, LOW = Input
PORTx
When OUTPUT: writes HIGH/LOW to pin. When INPUT: enables (1) or disables (0) internal pull-up resistor
PINx
Reads current state of the pin (read-only)
06 — I/O Functions

Digital & Analog I/O

Digital I/O (Arduino Functions)

pinMode(pin, mode);          // Sets pin as INPUT or OUTPUT
                             // Writes 1 bit in DDRx register

digitalWrite(pin, value);    // Sets pin HIGH (1) or LOW (0)
                             // Writes 1 bit in PORTx register

int val = digitalRead(pin);  // Reads pin value (0 or 1)
                             // Reads 1 bit from PINx register

Analog Input (ADC)

Analog input pins: A0 to A5. The ATmega328P has a 10-bit ADC (1024 steps).

int val = analogRead(pin);   // Returns 0–1023
                             // Converts 0–5V → 10-bit number
ADC Resolution Formula Resolution = VREF ÷ 2n = 5V ÷ 1024 ≈ 4.88 mV per step (with 5V AREF, 10-bit ADC)

Analog Output (PWM)

Analog output pins: 3, 5, 6, 9, 10, 11 (PWM-capable digital pins). Operates at ~490 Hz by default.

analogWrite(pin, value);     // value: 0–255 (8-bit)
                             // 0 = 0% duty cycle (always LOW)
                             // 255 = 100% duty cycle (always HIGH)
                             // 127 ≈ 50% duty cycle

Serial Communication Library (Arduino)

Serial.begin(baud_rate);     // Initialize UART (9600 default)
Serial.println(string);      // Print with newline
Serial.print(string);        // Print without newline
int b = Serial.read();       // Read one byte (buffered)

Time Functions

delay(ms);                   // Delay in milliseconds
delayMicroseconds(us);       // Delay in microseconds
07 — AREF

AREF — Analog Reference Voltage

AREF is the reference voltage that the ADC uses to convert an analog signal to its corresponding digital value. It defines what voltage = 1023 (100%).

Types of AREF

Internal (Default)
5V on Arduino Uno. Code: analogReference(DEFAULT);
Internal 1.1V
Internal bandgap reference of 1.1V. Higher resolution for small signals.
External
Supply your own voltage (min 1.1V) to the AREF pin. Code: analogReference(EXTERNAL);
External GND must connect to Arduino GND.

Connection of AREF (Hardware)

  • Pin 21 (AREF) must be decoupled
  • Connect a 100nF capacitor between AREF and the neighboring GND pin, as close as possible to the chip

Resolution Comparison

AREF ValueResolution per StepUse case
5V (default)5 ÷ 1024 ≈ 4.88 mVGeneral 0–5V sensors
3.3V (external)3.3 ÷ 1024 ≈ 3.22 mV3.3V logic sensors
1.1V (internal)1.1 ÷ 1024 ≈ 1.07 mVPrecision low-voltage sensing
Exam Tip With 5V AREF: measuring 3V gives analogRead() = (3/5) × 1023 ≈ 614. With 3.3V AREF: measuring 3V gives (3/3.3) × 1023 ≈ 931 — much better resolution for 3V range signals.
08 — Interrupts

Polling vs. Interrupts

Polling

Repeatedly reading a data port and testing the input value at regular intervals, regardless of whether the value changed.

  • Simple to code
  • Predictable, guaranteed time slices for each device
  • Inefficient — CPU wastes cycles checking when nothing has changed

Interrupt

A signal generated by hardware or software that triggers the Interrupt Service Routine (ISR) to run. Occurs only when needed.

Interrupt Flow

How an Interrupt Works — Step by Step

  • 1. Interrupt signal arrives (hardware pin change, timer overflow, UART byte received, etc.)
  • 2. Current instruction completes
  • 3. Return address (PC) is pushed onto the Stack
  • 4. CPU jumps to the ISR (Interrupt Service Routine) via the Interrupt Vector Table
  • 5. ISR executes
  • 6. Return address is popped from stack — original program resumes

Types of AVR Interrupts

CategoryExamples
Pin change (external)Rotary encoder, button press, infrared sensor, keypad
Timer interruptsTimer overflow, output compare match
UARTByte received / ready to transmit
SPIData transfer complete
I²C (TWI)Address match, data received
ADCConversion complete
EEPROMReady for use
WatchdogTimer timeout (system recovery)

PCMSKx — Pin Change Mask Register

Each bit in PCMSKx corresponds to one PCINT pin in the group. Setting a bit to 1 enables that specific pin to trigger the interrupt.

  • PCMSK0 — controls PCINT0 to PCINT7 (Port B)
  • PCMSK1 — controls PCINT8 to PCINT14 (Port C)
  • PCMSK2 — controls PCINT16 to PCINT23 (Port D)

Steps to Enable a PCINT

// Step 1: Enable global interrupts
sei();

// Step 2: Enable the PCINT group (e.g., group 2 for Port D)
PCICR |= (1 << PCIE2);

// Step 3: Enable specific pin (e.g., PD4 = PCINT20)
PCMSK2 |= (1 << PCINT20);

// Step 4: Define the ISR for the group
ISR(PCINT2_vect) {
    // Determine which pin changed, then handle
}

12 — Non-Volatile Memory

Non-Volatile Memory Types

ROM
Read-Only Memory. Data programmed at manufacture. Cannot be rewritten. Used for permanent system software.
MROM
Mask ROM. Data burned into chip during manufacturing. Only economical for mass production. Cannot be reprogrammed ever.
PROM
Programmable ROM. Blank when manufactured. Programmed once using a PROM Programmer by "blowing" fuses — irreversible. Invented 1956 by Wen Tsing Chow.
EPROM
Erasable PROM. Erased using UV light through a quartz glass window on the chip top. Erasure takes ~20 minutes. Requires 12.5V (Vpp) for programming. Removed from board to erase.
EEPROM
Electrically Erasable PROM. Written/erased electrically while on the circuit board. Byte-level erasure. Up to 100,000 write/erase cycles. ATmega328 has 1KB EEPROM.
Flash
Flash EEPROM. Entire device erased at once (in a "flash") — under 1 second. Block-level erasure possible. Intermediate write speed. ATmega328 has 32KB Flash. Used in USB drives, SD cards, SSDs.

13 — RAM

RAM — SRAM & DRAM

SRAM — Static RAM

  • Uses bistable flip-flop latching circuitry — 6 transistors per bit cell
  • Static — no refresh needed; data held as long as power is on
  • Faster access, lower latency
  • More complex circuit → less density per chip area
  • Used for: CPU cache, MCU on-chip RAM
  • ATmega328P: 2KB SRAM

DRAM — Dynamic RAM

  • Stores each bit as charge in a capacitor — 1 transistor + 1 capacitor per cell
  • Dynamic — capacitors discharge → needs periodic refresh (every few ms)
  • Simpler circuit → higher density, larger capacity
  • Slower than SRAM
  • Used for: Main system RAM (DDR4, DDR5)

SRAM vs DRAM Comparison

#SRAMDRAM
1Complex circuitry → less capacity per sizeSimple cell → high density/capacity
2Transistors store dataCapacitors store data
3Lower access time (faster)Higher access time (slower)
4Medium/more power consumptionReduced power consumption
5Low packing densityHigh packing density
6Cache memory (1MB–16MB)Main memory (4GB–16GB)
7On-chip (inside processor)Off-chip (on motherboard)
8CostlierCheaper
Virtual Memory When physical DRAM runs out, the OS creates virtual memory by temporarily transferring inactive data from DRAM to disk (swap space). This allows running programs larger than physical RAM.
14 — ATmega328P Memory

ATmega328P Memory Map

ATmega328P Memory Organization

Flash Program Memory Details

  • 32KB in-system reprogrammable Flash — at least 100,000 write/erase cycles
  • Organized as 16K × 16 bits (16-bit wide words)
  • Program Counter (PC) is 14-bit wide (2¹⁴ = 16,384 locations)
  • Two sections: Application and Boot Loader
  • AVR Bootloader allows Arduino IDE to write code to flash via USB (UART)

EEPROM in ATmega328P

  • 1KB (1024 bytes) of EEPROM — separate address space
  • Single byte read/write operations
  • At least 100,000 write/erase cycles
  • Used for non-volatile storage of constants, calibration data, settings
  • Requires special write sequence; accessed via EEAR, EEDR, EECR registers

Timer/Counter Overview

ATmega328P has 3 hardware timers — fully independent counting units:

TimerBitsMax CountPWM OutputsNotes
Timer08-bit255OC0A (PD6/pin 6), OC0B (PD5/pin 5)Used by delay() and millis()
Timer116-bit65535OC1A (PB1/pin 9), OC1B (PB2/pin 10)Best for precise timing / servo control
Timer28-bit255OC2A (PB3/pin 11), OC2B (PD3/pin 3)Can use asynchronous clock (external 32kHz)

Timer Operating Modes

Normal Mode
Counter counts up from 0 to MAX (255 or 65535), then overflows back to 0. Generate interrupts on overflow. Most common for simple timing/counting.
CTC Mode
Clear Timer on Compare — counter resets to 0 when it matches OCRnA. Frequency precisely controllable by changing OCRnA value. Use for audio frequency generation, adjustable time base.
Fast PWM
Counter counts 0→MAX, then resets. Output set HIGH at bottom, cleared on compare match. Single slope. Higher frequency.
Phase Correct PWM
Counter counts 0→MAX→0 (dual slope). Lower frequency but symmetrical. Used for motor control.

Clock Sources for Timer

  • Internal clock — system clock or prescaled: fCLK/8, fCLK/64, fCLK/256, fCLK/1024
  • External clock — applied to T0/T1 pin; sampled every system clock cycle; cannot be prescaled; max frequency < fCLK/2.5
Timer Frequency Formula ftimer = fCLK ÷ Prescaler   |   fOVF = fCLK ÷ (Prescaler × 256) [8-bit Normal Mode]

17 — PWM

PWM — Pulse Width Modulation

PWM is a technique where the width of a pulse is varied while keeping the frequency constant. The average output voltage is proportional to the duty cycle.

Key Definitions

Frequency (f)
Number of complete cycles per second. f = 1/T (Hz)
Period (T)
Time for one complete cycle. T = 1/f. Example: 50Hz → T = 20ms
Duty Cycle (D)
Fraction of the period when the signal is ON (HIGH). D = TON/(TON+TOFF) × 100%
Duty Cycle D (%) = TON / T × 100  |   Example: T=10ms, TON=2ms → D = 20%

PWM Duty Cycle Visualization

PWM Signals at Different Duty Cycles

Applications of PWM

  • Dimming LEDs / controlling light intensity
  • Providing analog output voltage (0–5V equivalent)
  • Generating audio signals
  • Variable speed control for DC motors
  • Generating modulated signals for IR remote controls
  • Servo motor control (20ms period, 1–2ms pulse width)

18 — SPI

SPI — Serial Peripheral Interface

SPI is a synchronous serial communication protocol that uses a shared clock line. Developed by Motorola in the mid-1980s.

SPI Lines (4-wire)

SCK
Serial Clock — generated by master; synchronizes all data transfer
MOSI
Master Out Slave In — data from master to slave
MISO
Master In Slave Out — data from slave to master
SS / CS
Slave Select / Chip Select — active LOW; selects which slave is active

SPI Topology

SPI Master–Slave Connection (Multiple Slaves)

SPI Characteristics

  • Full-duplex — simultaneous send and receive
  • Master controls the clock — no baud rate negotiation needed
  • No addressing — slaves selected by dedicated SS pin (pulled LOW)
  • Faster than I²C — up to tens of MHz
  • Requires more pins for multiple slaves (one SS per slave)
  • Point-to-point or Point-to-multipoint
  • Master in MCU; slaves can be sensors, ADC, LCD, DAC, Flash memory

19 — I²C

I²C — Inter-Integrated Circuit

I²C (pronounced "I-squared-C" or "I-two-C") was developed in the early 1980s by Philips Semiconductors to connect a CPU to peripheral chips in TV sets. It uses only 2 wires.

I²C Bus Lines

SDA
Serial Data Line — carries all data
SCL
Serial Clock Line — generated by master
Pull-up Resistors Required Both SDA and SCL must be connected to positive supply (VCC) via pull-up resistors (typically 4.7kΩ). Devices only pull lines LOW — never drive them HIGH.

I²C Key Features

  • Only 2 wires (vs 4 for SPI)
  • Multi-master, multi-slave — multiple masters can share the bus
  • 7-bit or 10-bit addressing → up to 128 or 1024 devices on one bus
  • Speed: standard 100 Kbps, fast mode 400 Kbps, fast-plus 1 Mbps
  • Short distance, on-board communication (ideal for PCB)
  • New devices can be added simply by connecting to the 2 bus lines
  • Combines best of SPI (multi-slave) and UART (simple wiring)

I²C Message Structure

I²C transfers data in "messages" broken into "frames". Each message consists of:

START
Address (7-bit)
R/W
ACK
Data (8-bit)
ACK/NACK
...
STOP
START condition
SDA goes LOW while SCL is HIGH — signals beginning of transmission
Address frame
7-bit slave address + 1 R/W bit (8 bits total)
R/W bit
0 = Master writes to slave; 1 = Master reads from slave
ACK/NACK
ACK (0): slave received correctly. NACK (1): error or end of transfer
Data frames
8 bits each, followed by ACK/NACK
STOP condition
SDA goes HIGH while SCL is HIGH — signals end of transmission

  • Start bit: always logic 0 (LOW) — signals start of frame
  • Data bits: 5–9 bits (usually 8 bits = 1 byte), LSB first
  • Parity bit: optional error checking (odd/even)
  • Stop bit(s): always logic 1 (HIGH); 1 or 2 bits — signals end of frame

Baud Rate

Baud rate = bits per second. Common values: 9600, 19200, 38400, 57600, 115200 bps. Both devices must agree on the same baud rate.

RS-232 Standard

  • First introduced in 1960 by Electronic Industries Association (EIA)
  • "Recommended Standard 232" — serial communication at medium distances
  • Voltage levels: +3V to +15V = logic 0 (SPACE); −3V to −15V = logic 1 (MARK)
  • TTL levels (Arduino): 0V = logic 0, 5V = logic 1 — must convert with MAX232 IC
  • Minimum connections: TX, RX, GND (3 wires)
  • Full implementation: up to 14 wires (with handshake signals)
  • Connector: DE-9 (9-pin D-sub); older standard used DB-25

MAX232 — Level Converter

The MAX232 IC converts TTL voltage levels (0/5V) from the ATmega to RS-232 voltage levels (±12V) required by the RS-232 standard. It is required when connecting ATmega directly to a PC COM port.

  • ATmega TX (5V TTL) → MAX232 → RS-232 TXD (±12V)
  • RS-232 RXD (±12V) → MAX232 → ATmega RX (5V TTL)
  • Arduino uses USB-to-UART bridge IC (e.g., CH340, FTDI) instead of MAX232
