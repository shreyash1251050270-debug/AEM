ATmega328P — A RISC Microcontroller

The ATmega328P is based on the AVR RISC architecture developed by Atmel (now Microchip). It features:
  • 131 powerful instructions, most executed in a single clock cycle
  • 32 × 8-bit general-purpose registers
  • Harvard architecture (separate program and data memory)
  • Two-stage instruction pipeline for efficient execution
  • Achieves throughput approaching 1 MIPS per MHz (Million Instructions Per Second)

Question 02 5 Marks

Explain the function of the ALU and Program Counter in ATmega328P.

ALU (Arithmetic Logic Unit)

Definition The ALU is the computational core of the ATmega328P. It performs arithmetic and logical operations on data stored in the 32 general-purpose registers.

Functions of ALU:

  • Arithmetic Operations: Addition (ADD), subtraction (SUB), increment (INC), decrement (DEC), multiply (MUL).
  • Logical Operations: AND, OR, XOR, NOT, bit shift, bit set/clear.
  • Comparison: Compare (CP) instructions that set status flags without changing data.
  • Status Register (SREG) Update: After each operation, the ALU updates flags — Zero (Z), Carry (C), Negative (N), Overflow (V), Half Carry (H), Sign (S).
  • In the ATmega328P, the ALU operates directly on the 32 registers, and most ALU operations execute in 1 clock cycle.

Program Counter (PC)

Definition The Program Counter is a special register that holds the address of the next instruction to be fetched from Flash program memory and executed.

Functions of Program Counter:

  • Sequential Execution: After each instruction fetch, the PC is automatically incremented to point to the next instruction.
  • Branching: For branch/jump instructions (JMP, CALL, RJMP), the PC is loaded with the target address.
  • Interrupt Handling: When an interrupt occurs, the current PC value is saved on the stack, and the PC is loaded with the interrupt vector address.
  • Subroutine Calls: CALL pushes PC onto the stack; RET pops it back.
  • In ATmega328P, the PC is 14 bits wide, capable of addressing up to 16K word locations (32 KB of Flash memory).

Question 03 5 Marks

Illustrate the different types of ports in ATmega328P. Elaborate digital and analog I/O pins with examples.

Ports in ATmega328P

The ATmega328P has 3 I/O ports, each 8 bits wide (though not all pins may be available):

PortPinsArduino MappingFunctions
Port BPB0–PB7Digital pins 8–13 + XTALDigital I/O, SPI (MOSI, MISO, SCK, SS), PWM
Port CPC0–PC6Analog pins A0–A5 + RESETAnalog input (ADC), I2C (SDA, SCL), Digital I/O
Port DPD0–PD7Digital pins 0–7Digital I/O, UART (TX, RX), External Interrupts, PWM

Each port has 3 registers:

  • DDRx (Data Direction Register) — Sets pin as input (0) or output (1)
  • PORTx (Data Register) — Writes output value or enables internal pull-up
  • PINx (Input Pins Register) — Reads the current state of the pin

Digital I/O Pins

ATmega328P has 14 digital I/O pins (D0–D13 in Arduino). They can be configured as input or output.

Example — LED Blinking (Digital Output):

void setup() {
  pinMode(13, OUTPUT);    // Set pin 13 as output
}
void loop() {
  digitalWrite(13, HIGH); // Turn LED ON
  delay(1000);            // Wait 1 second
  digitalWrite(13, LOW);  // Turn LED OFF
  delay(1000);
}

Analog Input Pins

ATmega328P has 6 analog input pins (A0–A5, mapped to Port C: PC0–PC5). These are connected to the internal 10-bit ADC that converts analog voltage (0–5V) into a digital value (0–1023).

Example — Reading a Potentiometer (Analog Input):

void setup() {
  Serial.begin(9600);
}
void loop() {
  int sensorValue = analogRead(A0); // Read analog pin A0
  Serial.println(sensorValue);      // Print value (0-1023)
  delay(100);
}

Note:

Analog pins (A0–A5) can also be used as digital I/O pins (D14–D19) when analog functionality is not needed.

What is the difference between hardware and software interrupts? List external and internal interrupts in ATmega328P.

What is an Interrupt?

Definition An interrupt is a signal that temporarily halts the main program execution, causes the CPU to execute a special function called the Interrupt Service Routine (ISR), and then returns to the main program.

External Interrupts in ATmega328P

  • INT0 (PD2 / Digital pin 2) — Triggered by rising edge, falling edge, or level change.
  • INT1 (PD3 / Digital pin 3) — Same trigger options as INT0.
  • PCINT0–PCINT23 — Pin Change Interrupts on any of the 24 I/O pins (grouped into 3 ports: PCINT0–7, PCINT8–14, PCINT16–23).

Internal Interrupts in ATmega328P

  • Timer Interrupts: Timer0/Timer1/Timer2 overflow, compare match, input capture.
  • ADC Interrupt: ADC conversion complete.
  • UART Interrupts: USART TX complete, RX complete, Data Register Empty.
  • SPI Interrupt: SPI transfer complete.
  • TWI (I2C) Interrupt: I2C event (start, stop, data received).
  • EEPROM Ready Interrupt: EEPROM write operation complete.
  • Analog Comparator Interrupt: When comparator output changes.
  • Watchdog Timer Interrupt: Watchdog timer timeout.

Question 06 5 Marks

How does ATmega328P act as an I2C Master?

I2C (Inter-Integrated Circuit) Protocol

Definition I2C (I²C) is a synchronous, multi-master, multi-slave, serial communication protocol that uses only 2 wires: SDA (Serial Data) and SCL (Serial Clock). In ATmega328P, it is called TWI (Two-Wire Interface).

I2C Pins in ATmega328P:

  • SDA = PC4 (Analog pin A4) — Bidirectional data line
  • SCL = PC5 (Analog pin A5) — Clock line (generated by master)

How ATmega328P Acts as I2C Master:

  1. Generate START Condition: The master pulls SDA LOW while SCL is HIGH — signals the beginning of communication.
  2. Send Slave Address: Master sends a 7-bit slave address + 1-bit R/W (0=Write, 1=Read) on SDA, clocked by SCL.
  3. Wait for ACK: The addressed slave responds with an ACK (pulls SDA LOW) on the 9th clock pulse.
  4. Send/Receive Data: Data is transferred byte-by-byte. Each byte is followed by an ACK/NACK from the receiver.
  5. Generate STOP Condition: Master releases SDA from LOW to HIGH while SCL is HIGH — signals end of communication.

Arduino Example — I2C Master Writing to Slave:

#include <Wire.h>

void setup() {
  Wire.begin();        // Join I2C bus as Master
}
void loop() {
  Wire.beginTransmission(0x27); // Address slave 0x27
  Wire.write("Hello");          // Send data
  Wire.endTransmission();       // Send STOP
  delay(500);
}

Question 07 5 Marks

How is PWM generated using Timer/Counter modules?

What is PWM?

Definition PWM (Pulse Width Modulation) is a technique that generates a digital waveform with a variable duty cycle. The average voltage output is proportional to the duty cycle, allowing control of motor speed, LED brightness, etc.
Duty Cycle Formula Duty Cycle (%) = (ON time / Total Period) × 100 Average Voltage = V_supply × (Duty Cycle / 100) Example: 5V supply, 50% duty → V_avg = 2.5V

PWM Generation using Timers in ATmega328P:

  1. The Timer/Counter continuously counts from 0 up to a maximum value (TOP).
  2. An Output Compare Register (OCR) holds a threshold value.
  3. When the counter value < OCR → Output pin is HIGH.
  4. When the counter value ≥ OCR → Output pin is LOW.
  5. This creates a square wave with duty cycle = OCR/TOP × 100%.
  6. Changing the OCR value changes the duty cycle.

PWM Modes:

  • Fast PWM: Counter counts from 0 to TOP, then resets. Single-slope. Higher frequency.
  • Phase-Correct PWM: Counter counts up to TOP then back down to 0. Dual-slope. Symmetric output — better for motor control.

analogWrite(9, 64);   // 25% duty cycle
analogWrite(9, 255);  // 100% duty cycle (always HIGH)

Question 08 5 Marks

Explain the role of MOSI, MISO, SCK, and SS pins.

SPI (Serial Peripheral Interface)

Definition SPI is a synchronous, full-duplex, master-slave serial communication protocol. It uses 4 wires and is faster than I2C. Used for high-speed communication with SD cards, displays, sensors.
PinFull NameDirection (Master)ATmega328P PinFunction
MOSIMaster Out Slave InOutput (Master → Slave)PB3 (D11)Data line from master to slave. Master sends data, slave receives data on this line.
MISOMaster In Slave OutInput (Slave → Master)PB4 (D12)Data line from slave to master. Slave sends data, master receives data on this line.
SCKSerial ClockOutputPB5 (D13)Clock signal generated by the master. Synchronizes data transfer. Both devices sample data on clock edges.
SSSlave Select (Chip Select)OutputPB2 (D10)Active LOW signal. Master pulls SS LOW to select a specific slave for communication. Each slave has its own SS line.

How SPI Communication Works:

  1. Master pulls the SS pin LOW for the target slave.
  2. Master generates SCK clock pulses.
  3. On each clock pulse, one bit is sent on MOSI (master→slave) and simultaneously one bit is received on MISO (slave→master).
  4. After data transfer, master pulls SS HIGH to deselect the slave.

Advantages of SPI:

  • Full-duplex (simultaneous send and receive)
  • Very fast (up to 10 MHz on ATmega328P)
  • No addressing scheme needed (SS selects slave)
  • Simple hardware

Question 09 5 Marks

Summarize any 5 features of ATmega328P.

  1. 8-bit AVR RISC Architecture: 131 instructions, most single-cycle execution. 32 × 8-bit general-purpose registers. Achieves up to 20 MIPS at 20 MHz clock speed.
  2. Memory: 32 KB Flash (program memory), 2 KB SRAM (data memory), 1 KB EEPROM (non-volatile data storage). Harvard architecture with separate program and data buses.
  3. I/O Ports: 23 programmable I/O pins organized into 3 ports (B, C, D). Each pin can be configured as input or output. Internal pull-up resistors available.
  4. Built-in Peripherals:
    • 6-channel 10-bit ADC
    • 3 Timer/Counters (Timer0: 8-bit, Timer1: 16-bit, Timer2: 8-bit)
    • 6 PWM output channels
    • USART (serial communication)
    • SPI (Serial Peripheral Interface)
    • TWI / I2C (Two-Wire Interface)
    • Analog Comparator
  5. Low Power Modes: 6 sleep modes (Idle, ADC Noise Reduction, Power-save, Power-down, Standby, Extended Standby) for battery-powered applications. Operating voltage: 1.8V–5.5V.

Question 10 5 Marks

What is the use of the following in Arduino IDE? (Answer any 5)

a. void setup()

Called once when the Arduino board is powered on or reset. Used to initialize pin modes, serial communication, libraries, and variables. Code inside runs only once at startup.

void setup() {
  pinMode(13, OUTPUT);   // Initialize pin 13 as output
  Serial.begin(9600);    // Start serial at 9600 baud
}

b. void loop()

Called repeatedly in an infinite loop after setup() completes. Contains the main program logic — reading sensors, controlling actuators, making decisions. This function runs forever.

void loop() {
  digitalWrite(13, HIGH);  // Turn LED on
  delay(1000);
  digitalWrite(13, LOW);   // Turn LED off
  delay(1000);
}

c. pinMode(pin, mode)

Configures a specified digital pin as INPUT, OUTPUT, or INPUT_PULLUP. Must be called before using digitalRead() or digitalWrite() on that pin.

pinMode(7, INPUT);         // Set pin 7 as digital input
pinMode(13, OUTPUT);       // Set pin 13 as digital output
pinMode(2, INPUT_PULLUP);  // Input with internal pull-up

d. pulseIn(pin, value)

Reads the duration of a pulse (HIGH or LOW) on a digital pin, in microseconds. Commonly used with ultrasonic sensors (HC-SR04) to measure echo time.

long duration = pulseIn(echoPin, HIGH);
// Returns time in μs the echo pin was HIGH
// Distance = duration × 0.034 / 2 (in cm)

e. digitalRead(pin)

Reads the value from a specified digital pin — returns HIGH (1) or LOW (0). Used to read button states, switch positions, digital sensor outputs.

int buttonState = digitalRead(7);  // Read pin 7
if (buttonState == HIGH) {
  // Button is pressed
}

f. digitalWrite(pin, value)

Writes a HIGH or LOW value to a digital output pin. Used to turn LEDs on/off, activate relays, drive motors.

digitalWrite(13, HIGH);  // Set pin 13 to 5V (LED on)
digitalWrite(13, LOW);   // Set pin 13 to 0V (LED off)

Question 11 5 Marks

Explain the importance of the analog pin in ATmega328P. Does it have ADC? What is the resolution?

Importance of Analog Pins

The analog pins (A0–A5) allow the ATmega328P to interface with the real-world analog environment. While digital pins only detect HIGH/LOW (binary), analog pins can measure continuously varying voltages — essential for reading sensors like:

  • Temperature sensors (LM35, thermistor)
  • Light sensors (LDR / photoresistor)
  • Potentiometers (variable resistance)
  • Pressure sensors, moisture sensors
  • Current sensors (ACS712)

Does ATmega328P have ADC?

Yes! The ATmega328P has a built-in 10-bit Successive Approximation ADC with 6 channels (multiplexed through a single ADC module).

ADC Resolution

Resolution ADC Resolution = 10 bits Number of steps = 2¹⁰ = 1024 steps (values 0 to 1023) Voltage Resolution = V_ref / 1024 With V_ref = 5V: Resolution = 5V / 1024 = 4.88 mV per step Conversion: Voltage = (ADC_value × V_ref) / 1024

ADC Features in ATmega328P:

  • 6 multiplexed channels (ADC0–ADC5 on pins PC0–PC5 / A0–A5)
  • 10-bit resolution (output: 0–1023)
  • Conversion time: 13–260 μs
  • Selectable voltage reference: AVCC (5V), internal 1.1V, or external AREF
  • Free-running or single conversion mode
  • ADC interrupt on conversion complete
